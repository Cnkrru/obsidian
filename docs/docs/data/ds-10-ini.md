# 数据序列化

---
## INI 文件基础知识

---
### 1. 基本结构
- INI 文件由节（section）、键（key）和值（value）组成，使用 `[section]` 表示节。

### 2. 键值对
- 格式：`key = value`，等号两侧可以有空格，也可以没有。

### 3. 节的作用
- 节用于分组相关的键值对，避免键名冲突。

### 4. 注释
- 使用分号（;）或井号（#）开始注释，注释直到行尾。

### 5. 数据类型
- 所有值默认视为字符串，不区分数据类型。

### 6. 大小写敏感性
- 通常不区分大小写，但取决于解析器实现。

### 7. 使用场景
- 常用于配置文件，如Windows系统配置、应用程序设置等。

### 8. 示例
```ini
; 这是注释
[database]
host = localhost
port = 3306

[user]
name = admin
password = secret
```
