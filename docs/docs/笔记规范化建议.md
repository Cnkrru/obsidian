# 笔记规范化建议

## 1. 目录结构

### 1.1 顶级目录
- **backend**：后端相关笔记
- **data**：数据相关笔记
- **election**：选举相关笔记
- **frontend**：前端相关笔记
- **hardware**：硬件相关笔记

### 1.2 子目录
- 每个顶级目录下按技术/主题分类创建子目录
- 例如：`backend/C`、`backend/python`、`frontend/css` 等

## 2. 文件名规范

### 2.1 命名格式
- 采用 `类别-序号-标题.md` 的格式
- 例如：`c-1-数据存储.md`、`css-1-选择器.md`

### 2.2 命名规则
- **类别**：使用小写字母，如 `c`、`js`、`py`、`css`、`html` 等
- **序号**：使用阿拉伯数字，从 1 开始，保持连续
- **标题**：使用中文，简短明了，反映文件内容
- **扩展名**：统一使用 `.md` 扩展名

## 3. 元数据规范

### 3.1 元数据格式
每个 Markdown 文件开头必须包含以下元数据：

```yaml
---
title: "文件标题"
description: "文件描述"
date: YYYY-MM-DD
tags: [标签1, 标签2, ...]
sidebar: auto
---
```

### 3.2 元数据要求
- **title**：文件的完整标题
- **description**：文件的简短描述
- **date**：文件创建或更新的日期
- **tags**：文件的标签，用于分类和搜索
- **sidebar**：设置为 `auto`，自动生成侧边栏

## 4. 内容结构规范

### 4.1 基本结构
每个文件应包含以下结构：

1. **元数据**：文件开头的 YAML 元数据
2. **标题**：文件的主标题
3. **简介**：文件内容的简要介绍
4. **主要内容**：按照逻辑顺序组织的内容
5. **示例**：相关代码或示例（可选）
6. **总结**：内容总结（可选）

### 4.2 章节层次
- 使用 Markdown 标题层级（#、##、### 等）组织内容
- 章节标题应简短明了，反映章节内容

## 5. 代码格式规范

### 5.1 代码块
- 使用 Markdown 代码块语法
- 明确指定代码语言，如：
  
  ```c
  // C 代码
  ```
  
  ```python
  # Python 代码
  ```

### 5.2 代码风格
- 保持代码缩进一致
- 使用标准的代码风格
- 避免过长的代码行

## 6. 格式规范

### 6.1 分割线
- 使用 Markdown 标准分割线：`---`
- 避免使用长等号或其他非标准分割线

### 6.2 列表
- 使用 Markdown 列表语法
- 保持列表缩进一致

### 6.3 表格
- 使用 Markdown 表格语法
- 保持表格对齐

### 6.4 链接
- 使用 Markdown 链接语法
- 对于内部链接，使用相对路径

## 7. 内容规范

### 7.1 内容要求
- 内容应准确、清晰、简洁
- 避免冗余或重复内容
- 保持内容的一致性和连贯性

### 7.2 示例要求
- 示例应与内容相关
- 示例应简洁明了
- 示例应可执行（如果适用）

## 8. 版本控制

### 8.1 提交规范
- 提交消息应简洁明了
- 提交消息应反映更改的内容
- 避免一次性提交大量更改

### 8.2 分支管理
- 使用主分支（main）存储稳定版本
- 使用特性分支（feature）开发新功能
- 使用修复分支（fix）修复问题

## 9. 工具建议

### 9.1 编辑器
- 推荐使用 VS Code 或其他支持 Markdown 的编辑器
- 安装 Markdown 相关插件，如 Markdown All in One、Prettier 等

### 9.2 预览工具
- 使用编辑器的 Markdown 预览功能
- 或使用专门的 Markdown 预览工具

### 9.3 格式化工具
- 使用 Prettier 或其他代码格式化工具保持代码格式一致
- 使用 Markdown 格式化工具保持 Markdown 格式一致

## 10. 总结

通过遵循以上规范，可以使笔记更加整洁、规范、易于维护和阅读。同时，规范的笔记结构也有助于提高学习效率和知识管理能力。

建议在创建新笔记或修改现有笔记时，参考本规范，确保笔记的一致性和规范性。
